Best Practices

Follow these guidelines for optimal results with EasyForm and EasyList.

üí° Development Best Practices

Form Design

  • Keep forms as short as possible
  • Group related fields logically
  • Use clear, descriptive labels
  • Provide helpful placeholder text
  • Show validation errors clearly
  • Use appropriate input types

Security

  • Always validate on server-side
  • Use CSRF protection (built-in)
  • Sanitize all input data
  • Hash passwords properly
  • Validate file uploads carefully

Performance

  • Use pagination for large datasets
  • Implement server-side processing
  • Cache frequently accessed data
  • Optimize database queries
  • Use AJAX for better UX

User Experience

  • Provide clear feedback messages
  • Use loading indicators
  • Make interfaces responsive
  • Test on multiple devices
  • Follow accessibility guidelines

Frequently Asked Questions

Common questions and solutions for EasyForm and EasyList.

How do I handle file uploads securely? ‚ñº

EasyForm provides secure file upload handling with built-in validation:

$form->file('avatar', 'Profile Picture', [ 'accept' => 'image/*', 'maxsize' => '2MB', 'allowed_types' => ['jpg', 'jpeg', 'png', 'gif'], 'upload_path' => '/secure/uploads/', 'validate_image' => true ]);

Always validate file types on the server side and store uploads outside the web root when possible.

Can I integrate with existing frameworks? ‚ñº

Yes! EasyForm/EasyList work great with popular PHP frameworks:

  • Laravel: Use as a service or in controllers
  • Symfony: Integrate with Twig templates
  • CodeIgniter: Use in views and controllers
  • WordPress: Create plugins and admin interfaces

The libraries are framework-agnostic and use standard PHP, making integration straightforward.

How do I customize the styling? ‚ñº

EasyForm supports multiple CSS frameworks and custom styling:

// Use different themes $form = new EasyForm('my_form', [ 'theme' => 'bootstrap', // or 'semantic', 'material' 'class' => 'custom-form-class' ]); // Add custom CSS classes to fields $form->text('name', 'Name', [ 'class' => 'custom-input-class' ]);

You can also create completely custom themes by extending the base classes.

What about large datasets in EasyList? ‚ñº

EasyList handles large datasets efficiently with server-side processing:

$list = new EasyList('large_dataset'); $list->serverSide(true) ->ajax(['url' => '/api/data']) ->paginate(50) ->virtualScroll(['height' => 400]);

This loads data on-demand and only renders visible rows, handling millions of records smoothly.

How do I add custom validation rules? ‚ñº

You can add custom validation using callbacks or extend the validation system:

// Custom validation callback $form->rule('domain', function($value) { if (!preg_match('/^[a-zA-Z0-9\-]+\.[a-zA-Z]{2,}$/', $value)) { return 'Please enter a valid domain name'; } return true; }); // Or register global custom rules EasyForm::addValidationRule('strong_password', function($value) { return preg_match('/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]/', $value); });
