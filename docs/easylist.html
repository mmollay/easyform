EasyList Documentation

Master the powerful PHP data table generator with advanced features like search, filtering, sorting, pagination, and export capabilities.

What is EasyList?

EasyList is a powerful PHP library for creating dynamic, interactive data tables. It transforms raw data into beautiful, searchable, sortable, and exportable tables with minimal code.

Key Features

  • Fluent Interface: Chainable methods for clean code
  • Multiple Data Sources: Arrays, databases, APIs
  • Advanced Search: Global and column-specific filtering
  • Smart Sorting: Multi-column sorting with custom rules
  • Pagination: Efficient handling of large datasets
  • Export Options: CSV, Excel, JSON, PDF formats
  • Action Buttons: Custom row and bulk actions
  • Responsive Design: Mobile-friendly tables
  • AJAX Support: Real-time data loading
  • Customizable: Full control over appearance
Perfect For: Admin panels, dashboards, data reports, user management, product catalogs, and any application requiring data visualization.

Quick Start

Create your first data table in minutes with this simple example.

Basic Data Table

1, 'name' => 'John Doe', 'email' => 'john@example.com', 'role' => 'Admin', 'created_at' => '2024-01-15'], ['id' => 2, 'name' => 'Jane Smith', 'email' => 'jane@example.com', 'role' => 'User', 'created_at' => '2024-01-20'], ['id' => 3, 'name' => 'Bob Johnson', 'email' => 'bob@example.com', 'role' => 'Moderator', 'created_at' => '2024-01-25'] ]; // Create the list $list = new EasyList('users_table'); $list->data($users) ->column('id', 'ID', ['width' => '80px', 'align' => 'center']) ->column('name', 'Full Name', ['sortable' => true]) ->column('email', 'Email Address', ['searchable' => true]) ->column('role', 'Role', ['filter' => ['Admin', 'User', 'Moderator']]) ->column('created_at', 'Created', ['type' => 'date', 'format' => 'M j, Y']) ->searchable() ->paginate(10) ->exportable(['csv', 'excel']) ->display(); ?>

Custom Sort Functions

column('full_name', 'Full Name', [ 'sortable' => true, 'sortCallback' => function($a, $b, $direction) { // Sort by last name, then first name $lastNameA = explode(' ', $a['full_name'])[1] ?? ''; $lastNameB = explode(' ', $b['full_name'])[1] ?? ''; $result = strcmp($lastNameA, $lastNameB); return $direction === 'desc' ? -$result : $result; } ]) // Natural sorting for version numbers ->column('version', 'Version', [ 'sortable' => true, 'sortCallback' => function($a, $b, $direction) { $result = version_compare($a['version'], $b['version']); return $direction === 'desc' ? -$result : $result; } ]); ?>

Best Practices

Guidelines for creating efficient and user-friendly data tables.

Performance Optimization

  • Use pagination: Don't load thousands of rows at once
  • Server-side processing: For datasets > 1000 rows
  • Lazy loading: Load data on demand
  • Debounced search: Prevent excessive API calls
  • Virtual scrolling: For very large datasets

User Experience

  • Clear column headers: Make data purpose obvious
  • Appropriate data types: Use proper formatting
  • Consistent actions: Place action buttons logically
  • Loading indicators: Show progress during data loading
  • Error handling: Graceful degradation on failures

Code Organization

data($data) ->columns($this->getColumns()) ->actions($this->getActions()) ->filters($this->getFilters()) ->pagination($this->getPaginationConfig()); } private function getColumns() { return [ ['key' => 'id', 'label' => 'ID', 'width' => '60px'], ['key' => 'name', 'label' => 'Name', 'sortable' => true], ['key' => 'email', 'label' => 'Email', 'searchable' => true] ]; } private function getActions() { return [ 'edit' => ['icon' => 'edit', 'url' => '/users/edit/{{id}}'], 'delete' => ['icon' => 'trash', 'confirm' => true] ]; } } // Usage $builder = new UserTableBuilder(); $table = $builder->build($users); $table->display(); ?>

Security Considerations

  • Input validation: Validate all search and filter inputs
  • Permission checks: Verify user permissions for actions
  • Data sanitization: Escape output to prevent XSS
  • Rate limiting: Prevent API abuse
  • Sensitive data: Don't expose internal IDs or sensitive fields
