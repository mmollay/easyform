Installation Guide

Get EasyForm and EasyList up and running in your project quickly and easily. Choose between Composer installation or manual setup.

PHP Extensions

The following PHP extensions are required or recommended:

Required Extensions

  • json - JSON processing
  • mbstring - Multi-byte string handling
  • session - Session management

Optional Extensions

  • intl - Internationalization

Manual Installation

If you prefer not to use Composer or need more control over the installation, you can install EasyForm/EasyList manually.

Step 1: Download Files

Download the EasyForm/EasyList package and extract it to your project directory:

# Download from GitHub wget https://github.com/smartform/easyform/archive/main.zip unzip main.zip # Or clone the repository git clone https://github.com/smartform/easyform.git

Step 2: Copy Files to Your Project

# Copy the easy_form directory to your project cp -r easy_form/ /path/to/your/project/ # Your project structure should look like: /your-project/ ‚îú‚îÄ‚îÄ easy_form/ ‚îÇ ‚îú‚îÄ‚îÄ EasyForm.php ‚îÇ ‚îú‚îÄ‚îÄ src/ ‚îÇ ‚îÇ ‚îî‚îÄ‚îÄ EasyList.php ‚îÇ ‚îú‚îÄ‚îÄ autoload.php ‚îÇ ‚îú‚îÄ‚îÄ examples/ ‚îÇ ‚îî‚îÄ‚îÄ js/ ‚îú‚îÄ‚îÄ your-files.php ‚îî‚îÄ‚îÄ ...

Step 3: Include the Autoloader

Step 4: Set Up File Permissions

Ensure proper file permissions for web server access:

# Set appropriate permissions chmod -R 755 easy_form/ chmod -R 644 easy_form/*.php chmod -R 644 easy_form/src/*.php

Configuration

EasyForm/EasyList works out of the box with sensible defaults, but you can customize the configuration as needed.

Basic Configuration

'semantic', // semantic, bootstrap, material 'size' => 'large', // mini, tiny, small, medium, large, huge 'width' => 800, // Max width in pixels 'language' => 'en', // Language for validation messages 'ajax' => true, // Enable AJAX by default 'liveValidation' => true, // Real-time validation 'showErrors' => true // Show validation errors ]); ?>

Global Configuration

You can set global defaults for all forms in your application:

'bootstrap', 'size' => 'medium', 'language' => 'en', 'liveValidation' => true ]); // All forms will now use these defaults $form1 = new EasyForm\EasyForm('form1'); // Uses global defaults $form2 = new EasyForm\EasyForm('form2', ['theme' => 'semantic']); // Overrides theme ?>

Troubleshooting

Common installation issues and their solutions:

Class Not Found Errors

Error: Fatal error: Class 'EasyForm\EasyForm' not found

Solutions:

  • Verify the autoloader is included: require_once 'easy_form/autoload.php';
  • Check file paths are correct
  • Ensure files have proper read permissions
  • Verify namespace usage: use EasyForm\EasyForm;

CSS/JavaScript Not Loading

Issue: Forms look unstyled or features don't work

Solutions:

  • Include required CSS framework (Semantic UI, Bootstrap, etc.)
  • Include jQuery for AJAX functionality
  • Check browser console for 404 errors
  • Verify file paths in HTML includes

Permission Errors

Error: Permission denied when accessing files

Solutions:

# Fix file permissions chmod -R 755 easy_form/ chown -R www-data:www-data easy_form/ # On Linux/Apache chown -R _www:_www easy_form/ # On macOS/Apache

PHP Version Issues

Error: Syntax errors or features not working

Solutions:

  • Upgrade to PHP 7.4 or higher
  • Enable required PHP extensions
  • Check phpinfo() for current configuration
