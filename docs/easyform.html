Master the modern PHP form builder with fluent API, advanced validation, and beautiful responsive designs.

What is EasyForm?

EasyForm is a revolutionary PHP form builder that transforms how you create HTML forms. With its fluent interface and powerful features, you can build complex forms in minutes instead of hours.

Key Features

  • Fluent API: Chainable methods for intuitive coding
  • 50+ Field Types: Everything from basic inputs to rich components
  • Responsive Design: Mobile-first approach
  • Built-in Validation: Client and server-side validation
  • AJAX Ready: Seamless asynchronous form handling
  • CSRF Protection: Automatic security measures
  • Multiple Themes: Semantic UI, Bootstrap, Material Design
  • Layout System: Tabs, accordions, columns, groups
  • Internationalization: Multi-language support
  • Type Safety: Full PHP 8 type hints
Migration from Smart-Form: EasyForm provides a modern alternative to array-based form configurations with 90% less code and full IDE support.

Validation System

Comprehensive validation with both client-side and server-side rules.

Basic Validation Rules

text('username', 'Username') ->rule('username', 'required|minlength[3]|maxlength[20]|alphanumeric'); $form->email('email', 'Email Address') ->rule('email', 'required|email|unique[users,email]'); $form->password('password', 'Password') ->rule('password', 'required|minlength[8]|strength[medium]'); $form->password('confirm_password', 'Confirm Password') ->rule('confirm_password', 'required|matches[password]'); ?>

Available Validation Rules

Rule Description Example
required Field must not be empty 'required'
email Valid email address 'email'
minlength[n] Minimum length 'minlength[8]'
maxlength[n] Maximum length 'maxlength[50]'
numeric Must be numeric 'numeric'
alphanumeric Letters and numbers only 'alphanumeric'
matches[field] Must match another field 'matches[password]'
url Valid URL 'url'
date Valid date 'date'
unique[table,column] Unique in database 'unique[users,email]'

Custom Validation

text('domain', 'Domain Name') ->rule('domain', 'required|callback_validate_domain'); // Define custom validation function validate_domain($value) { if (!preg_match('/^[a-zA-Z0-9\-]+\.[a-zA-Z]{2,}$/', $value)) { return 'Please enter a valid domain name (e.g., example.com)'; } return true; } // Complex validation with multiple conditions $form->number('age', 'Age') ->rule('age', function($value) { if (empty($value)) return 'Age is required'; if ($value < 18) return 'You must be at least 18 years old'; if ($value > 120) return 'Please enter a valid age'; return true; }); ?>

Best Practices

Tips and recommendations for creating excellent forms.

Form Design

  • Keep it simple: Only ask for essential information
  • Group related fields: Use tabs or fieldsets for complex forms
  • Provide clear labels: Make field purposes obvious
  • Use appropriate field types: Email for emails, tel for phones, etc.
  • Add helpful placeholders: Show expected format examples

Validation

  • Validate early: Use live validation for immediate feedback
  • Clear error messages: Explain what's wrong and how to fix it
  • Server-side validation: Always validate on the server too
  • Progressive enhancement: Form should work without JavaScript

Performance

'semantic']); $form->text('name', 'Name')->email('email', 'Email'); // Good: Chain methods for cleaner code $form->text('first_name', 'First Name') ->text('last_name', 'Last Name') ->email('email', 'Email') ->submit('Submit'); // Good: Use appropriate validation $form->email('email', 'Email') ->rule('email', 'required|email'); // Combined rules ?>

Security

  • CSRF protection: Always enabled by default
  • Input sanitization: Validate and sanitize all input
  • File upload security: Validate file types and sizes
  • SQL injection: Use prepared statements for database queries
